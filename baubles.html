Getting started:

Note: If you are looking into data driven Baubles you don't need this. Click here.

First we need to add Baubles to the dev environment.

1. In your build.gradle add Jitpack to the project repositories.

repositories {

...

maven {

url 'https://jitpack.io'

}

}

2. Then add the Baubles dependency to the project dependencies.

dependencies {

compile 'com.github.lazyMods:Baubles:{version}:api'

}

Note: {version} is the latest version of Bauble for the Minecraft version that you are working on.

Example:

If you are working on Minecraft 1.16.4, the latest version of Baubles is 1.18.3.

See more in here.

How to create an Bauble:

Currently Baubles has two ways to create an Bauble.

1. Creating a new Item and implementing the IBauble interface.

public class BaubleTest extends Item implements IBauble {


public BaubleTest(Properties properties) {

super(properties);

}


@Override

public BaubleType getBaubleType() {

return {BaubleType};

}

}

Note: See BaubleTypes to know the types of Baubles that {BaubleType} can return.

Note: IBauble has more methods that gives you the hability to have more control over the Bauble. getBaubleType() is required to assign the Bauble to the correct slot.

Note: Check IBauble to understand more about it.

2. Data driven Baubles. (BETA)

WARN! Data-driven Baubles don't have a way to craft them.

If you are familiar with the new way that Minecraft is using .json files to give you more customization in your world, Baubles does the same by adding a way to add Baubles with .json files.

First you need to create a file in %GAMEDIR/baubles/. (The name doesn't matter and be sure that the extension is .json)

Below we have an example with all the parameters that we can use, but some of those are optional, as you will see in a bit.

{

"registryName": "ring_fire",

"type": "ring",

"glint": false,

"displayName": "Ring Fire",

"requireMod": [

"woodenutilities"

],

"tooltips": [

"Saves you from fire",

"Thats good right?"

],

"effects": [

{

"effectLevel": 4,

"effectRegistryName": "minecraft:fire_resistance"

},

{

"effectLevel": 4,

"effectRegistryName": "minecraft:luck"

},

],

"showEffectsTooltip": false

}

  • registryName (mandatory)

    - unique name used to differentiate the Baubles.

  • type (optional)

    - Bauble Type that defines the slot of the Bauble.

    - BaubleType.AMULET = "amulet".

    - BaubleType.RING = "ring".

    - BaubleType.BELT = "belt".

    - BaubleType.HEAD = "head".

    - BaubleType.BODY = "body".

    - BaubleType.CHARM = "charm".

    - NOTE: If you don't specify the Bauble Type or insert something that the isnÂ´t above, the game will automatically set the BaubleType to Trinket. This means that your Bauble will be able to fit in all the slots available.

  • glint (optional)

    - adds the enchantment effect to your Bauble.

  • displayName (mandatory)

    - The Bauble name in game.

  • requireMod (optional)

    - List of ids that your Bauble requires to be added. This is mostly needed when you want to add effects or recipes that depends on the other mods.

  • tooltilps (optional)

    - Add more information to Bauble.

  • effects (optional)

    - When your Bauble is equipped, the effects that you define will be added to the player.

  • showEffectsTooltip (optional)

    - If true, the Bauble Item will show the effects in the tooltip area.

Note: This params can be changed while the game is running! Type /reload and all the changes will be loaded.
